## 第一章 · 概述

Agent Memory Manager 是一个 **AI Agent 三层记忆管理微服务**。它把记忆管理从业务逻辑中解耦，提供自动化的长短期记忆注入接口。

### 三层记忆模型

```
Agent 记忆    → agents + skills          → system prompt（人设 + 技能）
User 记忆     → memory_fragments         → 语义检索注入（top-K）
Session 记忆  → sessions + messages      → 最近 N 轮对话上下文
```

每次调用上下文接口时，三层记忆自动聚合为一个结构化的 `Context`，可直接拼装为 LLM 消息列表。

### 核心功能

| 功能 | 说明 |
|------|------|
| Agent / Skill 管理 | 注册 Agent（人设 + 配置）、创建 Skill（关键词触发 + 提示片段）、绑定与解绑 |
| Session / Message 管理 | 创建会话、追加消息、查询历史 |
| User 记忆管理 | 画像编辑、记忆片段的向量检索 / 全文检索 / 手动添加 |
| LLM 增强（可选） | 自动摘要生成、用户记忆提取、向量嵌入，未配置时降级 |
| pgvector 语义检索 | 余弦相似度 + 全文检索双模式 |
| Python SDK | 面向 LangChain 等框架的 `MemoryClient` API |
| 调试控制台 | 浏览器端 `/debug`，Agent 配置 / 记忆检索 / 上下文预览 / LLM 日志 |

### ⏳ 即将实现

| 功能 | 状态 |
|------|:---:|
| Context Composition API（三层聚合拼接为消息列表） | 开发中 |

## 第二章 · 部署

### 2.1 Docker 一键启动

```bash
git clone https://github.com/NullWinters/AgentMemoryManager.git
cd AgentMemoryManager
docker compose up -d
```

启动后包含两个容器：

| 容器 | 端口 | 镜像 |
|------|:---:|------|
| `db` | 5432 (内) / 5433 (宿) | `pgvector/pgvector:pg16` |
| `api` | 8000 (内) / **8080** (宿，可用 `API_PORT` 改) | 本地构建 (FastAPI) |

### 2.2 验证部署

```bash
curl http://localhost:8080/api/v1/health
```

预期响应：

```json
{"status": "ok", "pgvector": true}
```

### 2.3 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DATABASE_URL` | `postgresql+asyncpg://memory:memorypass@db:5432/memorydb` | PostgreSQL 连接串 |
| `MEMORY_API_KEY` | `""` (空) | API 认证 Key，为空时跳过验证 |
| `API_PORT` | `8080` | Docker 宿主机映射端口（容器内仍为 8000） |
| `MAX_SESSION_TURNS` | `20` | 每 Session 返回消息轮数上限 |
| `USER_MEMORY_TOP_K` | `5` | 检索用户记忆时返回最大条数 |

## 第三章 · 快速入门

以下示例全部使用 `curl`（默认端口 `8080`）。

### 3.1 无 LLM 的规则模式（最简单）

**① 注册 Agent**

```bash
curl -X POST http://localhost:8080/api/v1/agents \
  -H "Content-Type: application/json" \
  -d '{
    "agent_id": "book_bot",
    "name": "书籍推荐助手",
    "persona": "你是书籍推荐助手，语气亲切、专业...",
    "config": {"max_session_turns": 20}
  }'
```

**② 创建 Skill 并绑定**

```bash
curl -X POST http://localhost:8080/api/v1/skills \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Python进阶书单",
    "trigger_keys": ["Python", "进阶", "书单"],
    "prompt_snippet": "当用户需要进阶Python书籍时，推荐《Effective Python》..."
  }'
```

响应中取得 `skill_id`，然后绑定：

```bash
curl -X POST http://localhost:8080/api/v1/agents/book_bot/skills/{skill_id}
```

**③ 创建 Session（自动创建 User）**

```bash
curl -X POST http://localhost:8080/api/v1/sessions \
  -H "Content-Type: application/json" \
  -d '{"agent_id":"book_bot","user_id":"zhang","session_id":"sess_01"}'
```

**④ 对话循环**

```bash
# 追加用户消息
curl -X POST http://localhost:8080/api/v1/sessions/sess_01/messages \
  -H "Content-Type: application/json" \
  -d '{"role":"user","content":"推荐一本进阶 Python 书"}'

# 查询消息历史
curl "http://localhost:8080/api/v1/sessions/sess_01/messages?limit=20"
```

### 3.2 启用 LLM（完整模式）

在注册 Agent 时配置 `config.llm` 和 `config.embedding`：

```bash
curl -X POST http://localhost:8080/api/v1/agents \
  -H "Content-Type: application/json" \
  -d '{
    "agent_id": "bot",
    "name": "智能助手",
    "persona": "你是智能助手...",
    "config": {
      "max_session_turns": 20,
      "llm": {
        "api_key": "sk-xxx",
        "base_url": "https://api.deepseek.com",
        "model": "deepseek-chat"
      },
      "embedding": {
        "api_key": "BBFTQJUKU...",
        "base_url": "https://ai.gitee.com/v1",
        "model": "Qwen3-Embedding-8B"
      }
    }
  }'
```

此后每次添加消息，系统自动：生成 embedding → 写入 `messages.embedding`；超阈值（41 条）时生成摘要 + 提取用户记忆。

### 3.3 使用 Python SDK

```bash
pip install -e sdk/
```

```python
from agentmemory import MemoryClient

client = MemoryClient(
    base_url="http://localhost:8080",
    api_key="dev-secret"
)

# 注册 Agent
client.setup(agent_id="bot", name="助手", persona="你是助手...")

# 开始会话
session = client.session_start(agent_id="bot", user_id="zhang")

# 对话
session.add_message(role="user", content="推荐一本书")
ctx = session.inject_context(query="推荐一本书")

# ctx.to_messages() → 可传给任意 LLM
messages = ctx.to_messages()

session.add_message(role="assistant", content="...")
session.end()
```

## 第五章 · LLM 配置指南

### 5.1 配置结构

`agents.config` 中有两个独立配置块：

```json
{
  "config": {
    "llm": {
      "api_key": "sk-xxx",
      "base_url": null,
      "model": "gpt-4o-mini"
    },
    "embedding": {
      "api_key": "sk-yyy",
      "base_url": null,
      "model": "text-embedding-3-small"
    }
  }
}
```

- **`llm`** — Chat 模型（用于会话摘要和用户记忆提取）
- **`embedding`** — 文本转向量模型（用于语义检索；**不配置时 fallback 到 `llm`**）

### 5.2 三种配置模式

| 模式 | `llm` | `embedding` | 行为 |
|------|:---:|:---:|------|
| 规则模式 | ❌ | ❌ | 无 LLM，仅存储消息 |
| 共享模式 | ✅ | ❌ (fallback) | Chat 和 Embedding 共用同一 Provider |
| **分离模式** | ✅ | ✅ | Chat 和 Embedding 各自独立 Provider |

### 5.3 支持的 Provider

任何兼容 OpenAI API 格式的服务均可使用。

| Provider | Chat | Embedding | base_url |
|----------|:---:|:---:|------|
| OpenAI | ✅ | ✅ | `https://api.openai.com` |
| DeepSeek | ✅ | ❌ | `https://api.deepseek.com` |
| Azure OpenAI | ✅ | ✅ | `https://xxx.openai.azure.com` |
| Ollama (本地) | ✅ | ✅ | `http://localhost:11434/v1` |
| GITEE AI (模力方舟) | ✅ | ✅ | `https://ai.gitee.com/v1` |

### 5.4 降级行为

| LLM 状态 | Messages | Embedding | Summary | Memory Extraction |
|----------|:---:|:---:|:---:|:---:|
| `llm.api_key` 为空 | ✅ 正常存储 | — | — | — |
| `llm.api_key` 有效 | ✅ | ✅ (如配置) | ✅ (≥41条) | ✅ (≥41条) |
| Embedding API 失败 | ✅ | NULL (降级) | ✅ | ✅ |
| Chat API 失败 | ✅ | ✅ | — | — |

### 6.3 Session

```python
session = client.session_start(agent_id="book_bot", user_id="zhang")

# 对话
session.add_message(role="user", content="推荐一本进阶 Python 书")
session.add_message(role="assistant", content="推荐《Fluent Python》")

# 获取上下文

# 结束
session.end()  # 或使用 with 语句
# with session as s:
#     s.add_message(...)
```

### 6.4 Context.to_messages() 拼接规则

```
[0]        system:  persona + skills + user profile
[1..K]     user:    "[历史记忆] {fragment.content}" × top-K
[K+1..]    会话消息 (最近 N 轮)
```

## 第九章 · 常见问题

### Q: 为什么消息发出后 `messages.embedding` 是 NULL？

A: 检查 Agent 的 `config.llm.api_key` 和 `config.embedding.api_key` 是否已配置且有效。另外，某些 Provider（如 DeepSeek）不支持 embedding API，embedding 会优雅降级为 NULL。

### Q: 为什么 `VECTOR` 列没有固定维度？

A: 不同 Provider 返回不同维度（OpenAI=1536, GITEE AI=1024）。系统使用无维度约束的 `VECTOR` 类型，兼容任意维度的 embedding。

### Q: 如何接入本地 Ollama 模型？

A: Agent 配置中设置 `base_url` 为 Ollama 地址：

```json
{
  "llm": {
    "api_key": "ollama",
    "base_url": "http://localhost:11434/v1",
    "model": "qwen2.5"
  },
  "embedding": {
    "api_key": "ollama",
    "base_url": "http://localhost:11434/v1",
    "model": "nomic-embed-text"
  }
}
```

### Q: 消息数达到多少时会触发摘要和记忆提取？

A: 当前阈值固定为 **40 条消息**。超过阈值后，每次添加消息都会在后台生成摘要和提取记忆。

### Q: 调试前端连接不上怎么办？

A: 确认 `docker compose up -d` 已启动，访问 `http://localhost:8080/debug`（注意端口是 8080，不是 8000）。确保已在页面状态栏配置正确的 API 地址和 API Key。
